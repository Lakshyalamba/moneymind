# MoneyMind - Project Explanation and Context

## Overview
MoneyMind is a comprehensive personal finance web application. It enables users to track income and expenses, manage savings goals, view detailed dashboard analytics, and interact with an AI-powered financial assistant that provides insights based on the user's specific transaction history. 

## Tech Stack
**Frontend:**
- **Framework:** React.js powered by Vite.
- **Routing:** React Router DOM.
- **Charts/Analytics:** Recharts.
- **Styling:** Vanilla CSS and React Icons.

**Backend:**
- **Runtime:** Node.js.
- **Framework:** Express.js.
- **Database ORM:** Prisma.
- **Database:** PostgreSQL.
- **Authentication:** JWT (JSON Web Tokens) with HttpOnly cookies, bcryptjs for password hashing.
- **AI Integration:** Google Gemini API (with a local fallback mechanism if the API key is missing or the service is down).

## Dependencies & Libraries

### Frontend Dependencies (`frontend/package.json`)
- `axios` (^1.13.2) - Used for making HTTP requests (though `fetch` may be predominantly used as per internal utility functions).
- `react` / `react-dom` (^18.2.0) - Core UI library.
- `react-icons` (^5.5.0) - Provides the scalable vector icons used across the UI.
- `react-router-dom` (^6.8.1) - Manages multi-page routing and navigation.
- `recharts` (^3.4.1) - Renders the responsive charts (pie charts, line charts) on the dashboard.
- *Dev Dependencies:* `vite` (^4.1.0), `@vitejs/plugin-react` (^3.1.0).

### Backend Dependencies (`backend/package.json`)
- `express` (^4.18.2) - Web framework handling API routing and middleware.
- `@prisma/client` & `prisma` (^5.22.0) - Database ORM for managing PostgreSQL connections, schema generation, and migrations.
- `jsonwebtoken` (^9.0.2) - Generates and verifies secure JWTs for user authentication sessions.
- `bcryptjs` (^2.4.3) - Hashes and compares user passwords securely.
- `cookie-parser` (^1.4.7) - Parses `Cookie` header and populates `req.cookies` with the JWT token.
- `cors` (^2.8.5) - Enables Cross-Origin Resource Sharing with the frontend origin and allows passing credentials.
- `@google/generative-ai` (^0.21.0) - Official SDK to communicate with Google's Gemini models for the AI Financial Assistant chat feature.
- *Dev Dependencies:* `nodemon` (^3.0.2) - Used to hot-reload the backend server during development.

## Database Schema (Prisma)
The database structure relies on three main models:

1. **User Model:**
   - Fields: `id`, `name`, `email` (unique), `password`, `phone` (optional), `bio` (optional), `createdAt`, `updatedAt`, `refreshToken`.
   - Relationships: One-to-many with `Transaction` and `Goal`.

2. **Transaction Model:**
   - Fields: `id`, `amount` (Decimal), `type` (e.g., income, expense), `category` (e.g., Food, Salary, Utilities), `note`, `date`, `createdAt`, `updatedAt`.
   - Relationships: Belongs to a `User`.

3. **Goal Model:**
   - Fields: `id`, `title`, `targetAmount` (Decimal), `currentAmount` (Decimal, default 0), `deadline` (String), `createdAt`, `updatedAt`.
   - Relationships: Belongs to a `User`.

## Key Features & Capabilities

### 1. Authentication & Authorization
- Supports user Signup and Login.
- Employs a secure JWT strategy. Tokens are stored in HttpOnly cookies to prevent XSS attacks.
- Supports Token Refresh logic to keep the user session alive without needing to re-login frequently.
- Logout capability which clears the cookie and redirects the user.

### 2. Dashboard & Analytics
- Provides an immediate overview of total income, total expenses, and current balance.
- Displays recent transaction history.
- Utilizes Recharts to show visual breakdowns (e.g., pie charts for expense categories, line charts for income vs expense trends).
- Includes custom finance utility widgets on the dashboard for quick calculations (Currency Converter, SIP Calculator, EMI Calculator, FD Calculator, GST Calculator).

### 3. Transaction Management
- Users can create, update, and delete transactions.
- Provides views to filter, sort, and paginate through their transaction history.
- Differentiates between 'income' and 'expense' types, sorting them into various categories for accurate tracking.

### 4. Savings Goals
- Users can establish financial goals (e.g., "Buy a car", "Emergency Fund").
- Tracks the `targetAmount` against the `currentAmount`.
- Includes a deadline for achieving the goal.
- Users can update their progress towards the goal or delete it once completed.

### 5. Profile Management
- Users can view and edit their profile details (Name, Phone number, Bio).

### 6. AI Financial Assistant
- **Dedicated Chat Page:** A full-page chat interface where the user can ask for financial advice, budgeting tips, or insights on their spending habits.
- **Floating AI Widget:** A smaller, floating chat widget accessible across all authenticated pages for quick queries.
- **Gemini Integration:** The backend sends the user's transaction data alongside their prompt to Google's Gemini API to get personalized, context-aware financial advice.
- **Local Fallback:** If the Gemini API key is missing or fails, the application falls back to a locally defined algorithmic advice system so the chat feature remains functional.

## Project Structure
- `/backend`: Contains the Node/Express server, Prisma schema and migrations, and the REST API routes/controllers.
- `/frontend`: Contains the Vite/React frontend, including pages (Dashboard, Login, Signup, Chat, Goals, Transactions), reusable components (Sidebar, AIFloatingWidget), and styling.

## Summary for ChatGPT Prompts
When suggesting features, writing code, or debugging for this project, keep in mind:
- The backend relies heavily on Prisma for database interactions.
- Authentication relies on cookie-parser; you cannot access the JWT directly from `localStorage` on the frontend for API requests, rely on `credentials: 'include'` in fetch calls.
- The UI is styled with raw CSS (no Tailwind or UI libraries like Material UI are used heavily aside from icons and charts).
- Always ensure that new backend routes are protected using the existing authentication middleware if they access user-specific data.
